# 사운드 설계 — 전역 SFX 확장 + 합성 앰비언스 + 음소거 버튼

> **상태: 설계 확정.** 미결이던 말풍선 타자기 소리는 "하티/친구 두 음색으로 구분"으로
> 결정됐다. 다음 단계는 구현 계획(writing-plans).

## 배경

효과음 인프라는 이미 완성돼 있으나 미션 안에만 갇혀 있다.

- `src/scenes/planet/player/audio.ts` — 오실레이터 합성 SFX 13종
  (`tap` `pop` `select` `drop` `whoosh` `correct` `wrong` `stage` `sparkle`
  `recover` `reveal` `fanfare` `title`). 에셋 파일이 없어 APK 크기 영향 0.
- `MissionPlayer.tsx`에서만 24군데 호출. `player/` 바깥은 **전부 무음**
  (로그인·홈·행성 지도·프롤로그·씬 전환).
- `AudioManager.toggleMute()`는 존재하지만 **어디에서도 호출되지 않는다** — 음소거 UI가 없다.
- BGM(`src/lib/bgm.tsx`)은 곡 6개와 크로스페이드 재생기까지 완성돼 있으나
  `BGM_ENABLED = false`로 꺼져 있다. **어울리는 음원을 찾지 못해서**이며, 이 결정은 유지한다.

문제는 소리가 적은 게 아니라 한쪽에 몰려 있다는 것이다. 아이는 게임을 켜고 한참 무음을
겪다가 미션에 들어가는 순간 갑자기 소리 세계를 만난다.

## 설계 원칙

1. **소리는 피드백이거나 감정이거나, 둘 중 하나다.** 장식이면 뺀다.
2. **자주 일어나는 일일수록 짧고 작게.** 빈도와 길이는 반비례한다.
3. **같은 의미엔 같은 소리.** 기존 13종 안에서 해결한다. **예외는 타자기 blip 하나뿐**
   — "말하는 중"은 13종 중 어느 것과도 의미가 달라 대체할 게 없다(아래 참조).
4. **에셋을 추가하지 않는다.** 전부 합성 유지 → APK 0바이트.

## 왜 앰비언스가 필요한가

효과음은 조작할 때만 난다. 그러나 심심함을 느끼는 구간은 조작하지 않을 때다
(홈에서 행성 고르는 시간, 대사 읽는 시간, 문제 들여다보는 시간). 효과음만 늘리면
`조용함 → 삑 → 조용함` 이 되어 정적이 오히려 도드라진다.

합성 앰비언스는 멜로디가 없는 저음 패드다. 멜로디가 없으므로 **"이 게임에 어울리는가"를
판단할 필요 자체가 없어진다** — BGM이 막힌 원인(음원 탐색 실패)을 우회한다. 곡이 아니라
공간감이라 오래 들어도 질리지 않고 집중을 방해하지 않는다.

## 아키텍처

### AudioManager를 싱글턴으로 승격

현재 `MissionPlayer.tsx:149`가 `new AudioManager()`로 직접 인스턴스를 만든다.
→ 미션이 마운트돼야 오디오가 존재하고, `unlock()`도 미션 안 첫 탭에서만 걸린다.

변경:

- `src/scenes/planet/player/audio.ts` → **`src/lib/audio.ts`로 이동**, 앱 수명 동안
  하나만 사는 싱글턴을 export.
- `unlock()`은 **앱 전체의 첫 제스처**(App 레벨 `pointerdown`)에서 호출.
- `MissionPlayer`의 호출 24군데는 **그대로 두고 인스턴스 출처만 교체**한다.
  (검증된 미션 사운드를 건드리지 않는다.)

### 앰비언스

- `src/lib/ambience.ts` — `AudioManager`의 `ctx`/`master`를 공유하는 저음 패드 루프.
  오실레이터 2~3개를 아주 느린 LFO로 뜨고 지게 한다.
- 존 판별은 `bgm.tsx`의 `zoneForPath()` 로직을 재사용: `hub`(로그인·홈) /
  `planet`(행성·미션) / `silent`(인트로·아웃트로 — 동영상 자체 사운드).
- 존별로 패드의 기본 주파수만 다르게 한다.
- `BGM_ENABLED`와 같은 결로 **`AMBIENCE_ENABLED` 상수 하나로 끌 수 있게** 만든다.
  마음에 들지 않으면 되돌리는 비용이 한 줄이다.
- 음소거(`hg_muted`)는 master gain을 통하므로 앰비언스도 함께 꺼진다.

### 음소거 버튼

- App 레벨 전역 고정 구석 버튼. **`App.tsx`의 `<HiddenMenu />`와 같은 자리**
  (`<Routes>` 바깥, `SceneTransitionProvider` 안)에 둔다 → 로그인·홈·미션 어디서든
  보이고 눌린다. 히든 메뉴가 이미 이 패턴의 선례다(`App.tsx:101`).
- `AudioManager.toggleMute()`를 호출하고 상태는 기존 `hg_muted`(localStorage)에 저장.
- FixedStage 안이 아니라 App 레벨이므로 미션(`player/`, 자체 무대)에서도 동작한다.

**좌상단에 두지 않는다.** `useCornerLongPress.ts:14`의 히든 메뉴 진입 제스처가
좌상단 `CORNER_PX = 200` 정사각형을 쓴다. 제스처는 두 손가락을 요구하므로 한 손가락
탭과 기술적으로 충돌하지는 않지만, 교사용 히든 제스처 자리에 아이가 반복해 누르는 버튼을
겹쳐두지 않는다. 구현 시 다른 구석을 고르고, 각 씬(로그인·홈·미션)의 기존 UI와 겹치지
않는지 브라우저에서 확인한다.

## 사운드 맵

새로 소리가 붙는 지점은 **9개**. 그중 4개(씬 전환, 앰비언스 2존, 타자기)가 체감의
대부분이고 나머지 5개는 작은 마감이다.

### 전역

| 순간                   | 소리      |
| ---------------------- | --------- |
| 씬 전환 (`useFadeNav`) | `whoosh`  |
| 앰비언스 hub / planet  | 합성 패드 |

`sceneTransition.tsx` 한 곳만 고치면 홈→행성, 행성→홈, 로그인→홈이 전부 커버된다.
그 결과 **행성 버튼 탭 / 프롤로그 시작 버튼 / 프롤로그 홈 버튼에는 따로 소리를 넣지 않는다**
— 전환음이 바로 이어져 겹치기 때문이다. 세 지점을 공짜로 해결한다.

### 로그인

| 순간                    | 소리    |
| ----------------------- | ------- |
| 학교·이름 선택, 버튼 탭 | `tap`   |
| 로그인 성공             | `title` |
| 로그인 실패             | `wrong` |

성공/실패는 현재 텍스트로만 알린다. 소리가 붙으면 인지가 빨라진다.

### 홈

| 순간                       | 소리  |
| -------------------------- | ----- |
| 메뉴바 열기, 학습목표 열기 | `pop` |
| 팝업 닫기                  | `tap` |

**잠긴 행성 탭에는 소리를 넣지 않는다.** `PlanetButton.tsx:35`가 `disabled`라 이벤트가
오지 않으며, 소리를 내려면 `disabled`를 풀어야 해 접근성이 나빠진다. 자물쇠 아이콘이
이미 이유를 전달하고 있다.

### 프롤로그

**소리 없음.** `PrologueTemplate.tsx`에는 타자기도 대사 진행도 없고 시작/홈 버튼 두 개뿐이며,
두 버튼 모두 씬 전환음이 담당한다.

(초안에서 "대사 진행 → `tap`"이라 적었으나 코드 확인 결과 그런 인터랙션이 없어 삭제했다.)

### 미션

| 순간                       | 소리                       |
| -------------------------- | -------------------------- |
| 말풍선 타자기 (`typeInto`) | `blip` — 하티/친구 두 음색 |

기존 24군데는 **변경 없음.** 이미 적절히 배치돼 있고, 손대면 검증된 것을 깨뜨리기만 한다.
타자기 소리의 상세는 아래 절 참조.

## 말풍선 타자기 소리 (확정)

### 현황

`MissionPlayer.tsx:408-428`의 `typeInto`는 30ms마다 `vm.text = txt.slice(0, ++i)`로
글자만 늘리고 `render()`만 호출한다. **타이핑 중 소리는 전혀 없다.** 타이핑이 끝난 뒤
사용자가 탭할 때 `tap`이 나는 게 전부다.

같은 결의 타자기가 세 군데에 각각 구현돼 있다:

- `MissionPlayer.tsx:408` `typeInto` — 메인 대사 (가장 중요)
- `planet4/CourageCompassStage.tsx:64-71` — 보상 교훈
- `planet2/HiddenEmotionStage.tsx:59` — 자체 타자기

### 왜 유력한 후보인가

"효과음은 조작할 때만 나므로 정적을 못 없앤다"는 이 문서의 전제에서 **타이핑은 유일한 예외**다.
조작이 아닌데도 소리를 낼 명분이 있다 — 게임이 말하는 중이고 아이는 읽고 있으며 지금은 무음이다.
한국어 40자 대사면 1.2초의 정적이고, 미션 내내 수십 번 반복된다. 게임에서 가장 자주 나오는
콘텐츠가 무음인 셈이다. 앰비언스보다 체감이 클 수도 있다.

### 반드시 지킬 것 (안 지키면 톱질 소음이 된다)

30ms 간격은 **초당 33회**다. 기존 `tap`(50ms, gain 0.11)을 그대로 붙이면 소리가 겹친다.

- **2~3글자마다 한 번만** 재생 (초당 10회 근처)
- **공백·문장부호는 건너뛴다** (자연스러운 리듬이 생긴다)
- 아주 짧고(~25ms) 아주 작게(**gain 0.04** — 기존 `tap`의 약 1/3)

### 새 소리 1종 — `blip`

기존 13종으로 안 되고 새 소리가 하나 필요하다. 원칙 3과 충돌하지만 "말하는 중"은
13종 중 어느 것과도 의미가 달라 대체할 게 없다 — 원칙 쪽이 틀렸다고 판단했다.
합성이라 APK는 0바이트 그대로이므로 원칙 4는 유지된다.

### 화자별 음색 — 하티 / 친구 둘로 구분 (확정)

- **하티**(가이드) — 낮고 차분한 음
- **친구**(감정을 말하는 외계인) — 조금 높은 음

`showLine`이 이미 `node.speaker === "hati"`로 나누고 있다(`MissionPlayer.tsx:530`).
**주파수만 바꾸면 되므로 비용이 거의 0이다.**

행성별 외계인까지 구분하는 안은 **범위 밖**으로 뒀다 — 행성별 `theme`에 음성 설정을
넣어야 해 범위가 커진다. 둘로 구분해보고 부족하면 그때 재검토한다.

### 적용 범위

`MissionPlayer.tsx:408`의 `typeInto`(메인 대사)에만 적용한다. 나머지 두 타자기는
호출 빈도가 낮고 성격이 달라 **이번 범위에서 제외**한다:

- `planet4/CourageCompassStage.tsx:64-71` — 보상 교훈(미션당 1회, 화자 없음)
- `planet2/HiddenEmotionStage.tsx:59` — 자체 타자기

셋을 공용 헬퍼로 합치는 리팩터는 이번 목표(사운드)에 필요하지 않으므로 하지 않는다.

## 범위 밖

- BGM 활성화 (`BGM_ENABLED`는 `false` 유지). 음원을 찾으면 별도로 재검토한다.
- **`blip` 외의 새 SFX 종류 추가** — 나머지는 기존 13종으로 해결한다.
- 오디오 에셋 파일 추가.
- **기존 미션 사운드 24군데 조정** — 타자기 `blip`만 새로 넣고 나머지는 손대지 않는다.
- 행성별 외계인 음색 구분 (하티/친구 둘로만 나눈다).
- 타자기 3곳을 공용 헬퍼로 합치는 리팩터.
- 볼륨 슬라이더 (음소거 on/off로 충분).

## 검증

소리는 자동화 테스트로 판단할 수 없다. **브라우저에서 실제로 들어보는 것이 필수**이며,
아래는 전부 청취 확인 항목이다.

- `AudioManager` 이동 후 미션 사운드 24군데가 전과 동일하게 동작 (회귀 확인).
- 앱 첫 진입 → 로그인 화면 첫 탭에서 오디오 unlock 확인.
- 씬 전환음이 전환 페이드와 타이밍이 맞는지 (`FADE_MS = 160`).
- **타자기: 겹침·톱질 소음이 없는지, 하티와 친구가 구분되는지, 긴 대사에서 거슬리지 않는지.**
  이 스펙에서 튜닝이 가장 필요한 부분이다(throttle 간격·gain).
- 음소거 버튼: 모든 씬에서 보이고 기존 UI와 겹치지 않으며, 껐을 때 SFX·앰비언스·타자기가
  모두 무음이 되고, 앱 재시작 후에도 유지되는지(`hg_muted`).
- 음소거 버튼이 좌상단 히든 메뉴 제스처(`CORNER_PX = 200`)를 방해하지 않는지.
- `AMBIENCE_ENABLED = false`로 되돌렸을 때 앰비언스만 사라지고 SFX는 정상인지.
