## 목표

최소 스타일로 동작 중인 감정 설명서 게임에 원본(`mytemp/행성2_미션1_게임.html`) 학생 모드의
**초록/금색 양장본 책** 아이덴티티를 입힌다. 방향은 사용자 확정 = **원본 느낌 재현**.

## 범위

- 수정: `src/scenes/planet/planet2/EmotionGuideStage.tsx`(구조 요소·클래스 추가), `EmotionGuideStage.css`(전면 재작성).
- **무변경**: `emotionGuide.data.*`, `emotionGuide.logic.*`, 엔진(`types.ts`/`runner.ts`), `MissionPlayer.tsx`, `mission01.json`, `index.tsx`.
- 가운데 카드(`.eg-panel`, 1632×1020) 구조 유지 — 그 안을 양장본으로 채운다.
- 새 이미지 에셋 없음: 원본처럼 순수 CSS(종이질감 radial-gradient, 이모지)만. 오프라인 APK 안전.

## 재현할 요소 (원본 HTML 대응)

- **카드 프레임**: 짙은 초록(#1c3817/#142910) + 금색(#a39063) 테두리 + 네 귀퉁이 황금 경첩(#d4af37) + 좌측 책등 스티치.
- **상단 HUD**: 이름 칩 **제거**(이름 입력 화면을 뺐으므로). 붉은 리본 책갈피("감정 알기" + `n / 10`) + 공감 에너지 바(금→에메랄드, 완료 문항 비례).
- **양면 종이 페이지**: 좌/우 컬럼을 크림색(#f7f3e3)+도트 텍스처+안쪽 그림자. 상단 인덱스 태그("감정 상황 n" / "해소 & 공감") + 페이지 번호(P.1, P.2).
- **왼쪽**: 상황 일지 박스(🧭 헤더 + 🌫️ 데코 + 제목/본문) + 감정 8버튼(감정별 파스텔 색 CSS 매핑, 선택 시 금색 링+scale).
- **오른쪽**: 잠금 오버레이(🔒) + 행동 6카드(스스로해소=에메랄드 / 바라는공감=앰버 배지) + 피드백 박스(💡 "마음 연구 선배의 공감 지혜", 초록 점선).
- **하단 버튼**: 비활성=스톤, 활성=에메랄드 그라데이션.

## 적응 결정 (확정)

1. **이름 칩 제거** — HUD는 리본 + 에너지 바만.
2. **감정별 색은 CSS 클래스로 매핑**(`eg-emo--{id}`) — 데이터(JSON)와 결과 payload는 clean 유지(색을 되돌리지 않음).
3. **에너지 바 = 완료 문항 수 비례**(원본과 동일 의미): `(step-1)/TOTAL_STEPS`.

## 검증

- 로직/데이터/엔진 단위테스트 무변경 → 그대로 통과(회귀 없음).
- `npx tsc -b` + `npm run lint` clean.
- 1280×800 런타임 스크린샷: 문항 1개 플레이(감정선택→행동선택→피드백)로 양장본 룩·에너지 채움 확인, 0 콘솔 에러.

## 구현 방식

CSS 위주 반복 작업이라 서브에이전트 TDD 파이프라인 대신 **스크린샷 기반 직접 반복 구현**(레포의 씬 개발 관례). 커밋은 의미 단위.

- 이름 입력 표지·완료 축하 화면(원본), 교사 대기록관, 게임 로직/데이터 변경, 새 이미지 에셋.
