# auth 심사용 계정 로그인 버튼 — 설계

- 날짜: 2026-07-22
- 대상 씬: `www/auth/` (auth 씬 한정)
- 배경: 앱 스토어 심사 리뷰어가 계정 입력 없이 바로 로그인해 앱을 둘러볼 수 있게 한다.

## 목표

로그인 폼 화면 우측 상단(현재 비어 있는 공간)에 **"심사용 계정"** 버튼을 두어,
클릭 한 번으로 지정된 심사 전용 계정으로 로그인 → 홈까지 진입한다.

## 자격증명 (확정)

로그인 폼에 그대로 입력하는 값으로 하드코딩한다.

| 필드 | 값 |
| --- | --- |
| 학교 이름 | `테스트` |
| 학년 | `3` |
| 반 | `3` |
| 번호 | `3` |
| 비밀번호(PIN) | `3333` |

- **출처**: `www/auth/script.js` 안에 상수로 하드코딩. `dev-config.js`(gitignore·clean/CI 빌드엔 부재)에
  의존하지 않는다 — 배포 웹·APK 어디서든 버튼이 동작해야 하므로.
- 위험 평가: 전용 심사/데모 계정이라 자격증명이 클라이언트에 노출돼도 수용 가능.

## 배치

- 위치: `renderForm()`의 `auth-form-wrap` 최상단 줄. 현재 `← 뒤로` 버튼만 왼쪽에 있다.
- 변경: 그 줄을 좌우 배치 행으로 만들어 **왼쪽 `← 뒤로` / 오른쪽 `심사용 계정`**.
  - 구현: `back` 버튼과 새 버튼을 `justify-content: space-between` 플렉스 행(예: `.auth-form-head`)으로 감싼다.
    기존 `.auth-back { align-self: flex-start }`는 유지되도록 래핑 방식만 조정.
- 노출 조건: **로그인 모드에서만**(`mode === "login"`). 가입 모드·계정 선택(chooser) 화면엔 없음.
  로그인 화면에서는 **항상 표시**(리뷰어가 즉시 찾도록).
- 화면 최상단 우측의 ☰ 메뉴 버튼(`.hg-menu-btn`, 무대 코너 `top:10 right:10`)과는 별개 영역이라 충돌 없음.

## 동작

클릭 시 일반 로그인과 동일 경로를 태운다 — 폼 입력·검증 UI를 거치지 않고 하드코딩 creds로 직행:

1. 심사 계정 값으로 학교 매칭(`matchSchool`) → `toCredentials` 로 creds 구성.
   (학교 목록이 아직 안 왔거나 매칭 실패 시 폼 에러와 동일하게 안내.)
2. `verify(creds)` → 성공 시 `audio.play("title")` → `showWelcome()` (기존 login submit 핸들러와 동일 흐름).
3. 이후는 기존 welcome → 홈 진입 로직을 그대로 사용.

- 재사용: 가능하면 기존 login-mode submit 로직을 공용 함수로 뽑아 버튼과 공유(중복 최소화).
  최소 변경을 원하면 submit 핸들러의 로그인 분기를 호출 가능한 형태로 리팩터.

## 스타일

- 새 버튼은 `뒤로`와 시각적으로 형제가 되도록 `.btn.ghost` 계열 재사용, 크기·라운드는 `.auth-back`과 맞춤.
- 텍스트 요소 규약(`white-space: pre-line` + `word-break: keep-all`)은 버튼 라벨이 짧아 실질 영향 없으나
  새 텍스트 요소이므로 관례상 적용 대상 확인.

## 범위 밖 (YAGNI)

- ☰ 메뉴 안 버튼 추가(초기 아이디어) — 폐기.
- 전 페이지 공통 적용 — 하지 않음(auth 한정).
- 심사 기간 게이팅/자동 숨김 — 하지 않음(항상 표시).

## 검증

- `npm run dev` + playwright MCP로 실제 실행: 로그인 화면 우측 상단 버튼 노출 → 클릭 → 홈 진입, 콘솔 에러 0.
- 가입 모드·chooser 화면엔 버튼이 없음을 확인.
