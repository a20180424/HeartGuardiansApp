# Intro Scene 설계

> 작성일: 2026-06-28
> 대상 파일: `src/scenes/Intro.tsx` (전면 교체), `src/scenes/Intro.css` (신규)
> 실행 환경: Galaxy Tab A9+ 가로 1280×800 CSS px, DPR 1.5, immersive 전체화면 APK

## 목적

앱 진입 첫 화면. `Intro_v2.mp4`(1920×1080, 16:9, ~31초)를 보여준 뒤,
영상의 **마지막 프레임을 그대로 타이틀 배경으로 재활용**하여 그 위에 `시작하기`
버튼을 띄운다. 별도 타이틀 이미지/에셋은 만들지 않는다.

## 흐름 & 상태

영상 1개를 두 단계로 사용한다.

- **`playing`**: `Intro_v2.mp4` 음소거 자동재생. `건너뛰기` 버튼 + "탭하여 소리 켜기" 힌트 표시.
- **`ended`**: 영상이 마지막 프레임에 멈춘 상태. 그 화면 위에 큰 `시작하기` 버튼 표시.
- `시작하기` 클릭 → `nav("/auth")`.

`ended` 진입 경로 두 가지:

1. **끝까지 재생** → `onEnded` 발생. 브라우저는 종료 시 마지막 프레임을 그대로
   유지하므로 별도 처리 없이 `ended` 상태로 전환만 한다.
2. **`건너뛰기` 클릭** → `currentTime = Math.max(0, duration - 0.1)` 로 seek 후
   `pause()`, 그리고 `ended` 상태로 직접 전환한다.
   (`duration`은 `onLoadedMetadata`에서 확보.)

## 레이아웃 & UI

- 풀스크린 컨테이너: `min-height: 100svh`, 높이 `100dvh`, 배경 `#000`.
- `<video>`: `position: absolute; inset: 0; width/height: 100%; object-fit: cover`.
  → cover 풀필. 16:9 소스를 16:10 화면에 채우며 좌우 ~70px 크롭 허용(사용자 합의됨).
- `건너뛰기`: 우상단. `env(safe-area-inset-top/right)` 여백 고려. 작은 고스트 버튼 + ⏭ SVG 아이콘.
- "탭하여 소리 켜기" 힌트: 하단 중앙 작은 pill. 음소거일 때만 표시. 은은한 펄스 애니메이션. 🔇 SVG 아이콘.
- `시작하기`: 화면 중앙 하단. 크고 둥근 캔디형 버튼(보라 계열) + ▶/로켓 SVG 아이콘. `ended`에서만 표시, 페이드인 + 은은한 idle 바운스.

## 오디오

- 시작: `muted` 자동재생(확실한 자동재생 보장).
- `playing` 중 영상 영역 탭 → `muted=false`, "탭하여 소리 켜기" 힌트 숨김.
- 버튼들은 `z-index`로 탭 레이어 위에 두고 `stopPropagation`으로 소리 토글과 분리.

## 버튼 구현 방침

- **CSS 버튼 + 인라인 SVG 아이콘** 조합. 풀 SVG 버튼/PNG 이미지 버튼은 쓰지 않는다.
- 이유: 모든 버튼을 공용 CSS 클래스(`.btn` 계열) 기반으로 두면, 추후 다른 Scene을
  채운 뒤 **앱 전체 UI 일관성 검토 시 클래스 몇 개만 고쳐 한 번에 통일** 가능.
- 초등학생 대상 → 크고 둥근 만화풍, 굵은 큰 글씨, 바닥 두꺼운 그림자 3D 누름 효과,
  높은 가독성/접근성.
- 아이콘은 이모지 대신 **인라인 SVG** (Android WebView 이모지 렌더링 편차 회피).

## 엣지케이스

- **자동재생 차단 대비**: muted 자동재생은 거의 항상 되지만, `play()` 거부 시
  탭 레이어가 `play()`도 호출하도록 묶어 한 번의 탭으로 재생+소리해제를 모두 커버
  (주로 개발용 일반 브라우저 fallback).
- `duration`이 아직 없을 때 `건너뛰기` 눌림 방지: 메타데이터 로드 전이면 seek 대신
  바로 `ended`로 전환하거나 버튼 비활성. (구현 단계에서 단순화 결정)
- seek 목표는 `Math.max(0, duration - 0.1)` (정확히 `duration` seek은 프레임 미표시 위험).
