# 히든 점프 메뉴 (교사 시연용) — 설계

날짜: 2026-07-17
관련: `src/scenes/planet/devJump.ts` (선행 작업 — 16개 점프 URL)

## 목적

교사가 수업 중 태블릿 APK에서 **16개 미션 위치 + 홈**으로 즉시 점프할 수 있는 히든 메뉴.
지금까지 점프 파라미터는 `import.meta.env.DEV` 뒤에 있어 프로덕션 APK에선 동작하지 않는다.
이 작업은 그 게이트를 **런타임 잠금 해제**로 바꾸고, 태블릿에서 쓸 수 있는 진입 수단을 만든다.

## 결정 사항

| 항목 | 결정 |
|---|---|
| 사용자 | 교사(시연·수업 운영). 학생 발견은 막아야 함 |
| 진입 제스처 | 좌상단 100×100px, **두 손가락 2초 롱프레스** |
| 2차 잠금 | **4자리 PIN**, 앱 실행당 1회만 |
| 열리는 범위 | **모든 화면** (홈·프롤로그·미션 도중 포함) |
| 진도 저장 | **현행 유지**(끄지 않음) + 메뉴에 경고 라벨 |
| 점프 방식 | URL + 강제 리마운트 |

## 배경 제약 (설계를 강제한 사실들)

1. **세션은 메모리에만 있다.** `src/lib/session.ts`의 `let current`. 자격증명만 localStorage에
   있고 profile/progress는 매 실행 시 새로 받는다.
   → **`location.reload()`를 쓰면 세션이 날아가 교사가 로그인 화면으로 튕긴다.** 금지.
2. **태블릿에 키보드가 없다.** immersive 키오스크 APK. 히든 "키"는 터치 제스처여야 한다.
3. **무대가 두 종류다.** FixedStage(1280×800)와 미션 엔진 `#stage`(1920×1200).
   오버레이는 둘 다 피해야 한다.
4. **react-router는 search만 바뀌면 컴포넌트를 재사용한다.** 각 행성의
   `useState(initialStage)`는 마운트 시 1회만 읽으므로, 같은 행성 안에서의 점프는
   그냥 `nav()`로는 화면이 안 바뀐다.

## 아키텍처

### 1. 잠금 store — `src/lib/hiddenMenu.ts` (신규)

모듈 레벨 상태 + 구독 훅.

```
let unlocked = false;
export function isUnlocked(): boolean
export function unlock(pin: string): boolean   // 성공 시 true
export function useUnlocked(): boolean         // 구독 훅
```

- PIN 출처: `import.meta.env.VITE_HG_MENU_PIN`, 미설정 시 기본값 상수.
- 앱 실행당 1회 = 모듈 변수로 충분. APK 재실행 = 새 웹뷰 = 자동 잠김.
  localStorage에 **저장하지 않는다** (영구 해제되면 학생에게 노출된다).
- PIN은 번들에 들어가므로 APK를 뜯으면 보인다. 위협 모델은 교실의 초등학생이지
  리버서가 아니므로 수용한다.

### 2. 게이트 교체 — `src/scenes/planet/devJump.ts` (수정)

```
const dev = import.meta.env.DEV;          // 기존
const enabled = import.meta.env.DEV || isUnlocked();   // 변경
```

- 개발 중(`vite dev`): 지금처럼 PIN 없이 URL 파라미터가 그대로 통한다.
- 프로덕션 APK: 잠금 해제 후에만 파라미터가 해석된다.
- APK는 주소창 없는 immersive 웹뷰라 URL 파라미터 자체가 학생 손에 닿지 않는다.
  실질 방어선은 제스처 + PIN이다.

### 3. 강제 리마운트 — `src/App.tsx` (수정)

`/planet/N` 라우트를 search로 keying하는 래퍼로 감싼다.

```tsx
function Keyed({ children }: { children: ReactNode }) {
  const loc = useLocation();
  return <Fragment key={loc.search}>{children}</Fragment>;
}
```

파라미터가 바뀌면 행성 컴포넌트가 리마운트되어 `useState(initialStage)`가 다시 읽힌다.
URL이 단일 진실 공급원으로 남아 브라우저 개발 중 주소 입력도 계속 통한다.

### 4. 오버레이 — `src/shared/components/HiddenMenu.tsx` (신규)

App 루트(`Routes` 바깥, `Router` 안쪽 — `useNavigate`가 필요)에 마운트.

- 여기는 변환이 안 걸린 **실제 뷰포트**라 FixedStage/`#stage` 좌표계와 충돌 없이
  `position: fixed`로 전역 오버레이를 얹을 수 있다.
- **CLAUDE.md 예외**: vh/vw 금지는 "무대 안 씬"에 대한 규칙이다. 이 오버레이는
  무대 밖 앱 크롬이므로 뷰포트 기준 단위를 쓴다. 패널은 고정 크기 +
  `max-width/max-height` + 넘치면 스크롤.

내부 3단계 상태: `닫힘 → PIN 패드 → 그리드`.

- 이미 해제된 세션(`isUnlocked()`)이면 PIN을 건너뛰고 바로 그리드.
- **개발 중(`import.meta.env.DEV`)에는 PIN 패드를 건너뛴다.** 개발자가 매번 PIN을
  치는 건 무의미하다 — DEV에서는 devJump 게이트도 어차피 열려 있다.

### 5. 제스처 감지

App 루트에서 `pointerdown` 캡처 리스너.

- 조건: 좌상단 100×100(실제 뷰포트 기준) 안에 포인터 **2개 동시** + **2초 유지**.
- 포인터가 임계값 이상 움직이거나 하나라도 떼면 취소.
- `preventDefault`는 발동 시점에만. 그 전엔 아무것도 삼키지 않는다.
- 미션 화면 좌상단의 "탐험 진행도" 버튼은 단일 탭만 반응하므로 충돌하지 않는다.
- 키보드 단축키 `Ctrl+Shift+J` 병행 — 제스처와 **완전히 동일한 흐름**을 연다
  (프로덕션에서도 등록되지만 PIN을 똑같이 요구한다. 태블릿엔 키보드가 없어
  실질적으로 개발용이다).

### 6. 그리드 (17칸)

행 = 행성1~4, 열 = 미션1 / 미션2 / 미션3 / 엔딩. 하단에 홈 버튼.
**네 행성 전부 같은 규칙**으로 균일하다:

| 열 | URL |
|---|---|
| 미션1 | `#/planet/N?m=1` |
| 미션2 | `#/planet/N?m=2` |
| 미션3 | `#/planet/N?m=3` |
| 엔딩 | `#/planet/N?m=3&end=1` |

홈 = `#/home`.

행성3은 미션3이 독립 시나리오가 아니라 미션2 미니게임의 stage2지만,
`devJump`의 `mission3 → mission2` 별칭이 이를 흡수한다. 세 칸 모두 다른 화면이다:

- `?m=2` → 3D 월드 stage1(연료 채우기), 스테퍼 2
- `?m=3` → 3D 월드 stage2(얼어붙은 마음 녹이기), 스테퍼 3
- `?m=3&end=1` → `p3_m2_end` 보상 화면

**행성3 특례는 메뉴에 없다.** 격자는 균일하다.

### 7. 경고 라벨

그리드 상단에 고정 표시:

> ⚠️ 엔딩에서 "우주선으로 이동"을 누르면 진도가 자동 저장됩니다

정확성 근거: 저장은 엔딩 노드 **도착**이 아니라 거기서 **버튼을 누를 때**
(`onExit` → `completePlanet`) 일어난다. 엔딩만 보고 메뉴로 빠져나가면 저장되지 않는다.

현재 `completePlanet`은 행성1·2·3에만 배선돼 있다(planet4는 `nav("/home")`만).
행성4 미션3·엔딩은 아직 골격이고, 완성 시 자동저장을 넣을 예정이므로
라벨은 행성 번호를 명시하지 않는다.

## 알려진 위험 (수용)

교사가 학생 계정으로 로그인한 태블릿에서 엔딩에 점프해 "우주선으로 이동"을 누르면
그 학생의 진도가 서버에 저장된다. 되돌리는 UI는 없다.
경고 라벨이 유일한 완화책이다 — 의도된 트레이드오프(교사가 일부러 진도를 넣어주는
용도로도 쓸 수 있다).

## 범위 밖

- 행성4 `completePlanet(4)` 배선 (행성4 미션3·엔딩 본작업에서 함께)
- 진도 되돌리기 / 시연 모드
- 프롤로그(`?m=0`) 칸 — 필요해지면 열 추가

## 검증

1. `tsc -b`
2. 브라우저(`vite dev`): 제스처 대신 `Ctrl+Shift+J`로 열어 17칸 전부 클릭 →
   `#stage`의 `node-<id>` 클래스로 도착 노드 확인. 선행 작업에서 쓴
   24조합 순회 스크립트를 재사용한다.
3. **같은 행성 안에서의 점프**(예: 행성2 미션1 → 행성2 엔딩)가 실제로 화면을
   바꾸는지 확인 — 리마운트 래퍼의 핵심 회귀 지점.
4. 점프 후 세션이 살아있는지 확인(홈으로 돌아가 로그인 화면으로 안 튕기는지).
5. 프로덕션 게이트: `vite build && vite preview`에서 잠금 전에는 URL 파라미터가
   무시되고, 제스처+PIN 해제 후에는 동작하는지.
6. 실기기 APK에서 두 손가락 롱프레스 제스처 확인(미션 중·3D 월드 중 포함).
