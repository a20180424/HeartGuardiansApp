# 전역 메뉴 버튼 + 팝업 (음소거 · 앱 종료)

- 날짜: 2026-07-21
- 브랜치/worktree: `feat/global-menu-button`
- 상태: 설계 확정

## 목표

게임 전체(전 페이지)에 **우측 상단 메뉴 버튼(☰)** 을 배치하고, 누르면 **팝업**이
뜬다. 팝업에는 딱 두 항목만 넣는다:

1. **음소거 토글** — 기존 독립 음소거 버튼을 이 팝업 안으로 이관
2. **앱 종료** — 새로 추가 (확인창 경유)

## 배경 · 현재 구조

- 순수 vanilla 무빌드 MPA. `www/` 아래 씬(페이지)마다 독립 폴더(`index.html`+`script.js`+`style.css`).
- **페이지 간 js/css 공유 금지.** 공통 블록(음소거·히든 메뉴·SFX·fitStage·세션 가드)은
  **페이지마다 복사**돼 있다(의도된 격리 — A 수정이 B에 영향 없게).
- 현재 음소거 버튼: 각 `script.js`의 "공통 블록 3-② 음소거 버튼" — `.mute-btn`,
  `document.body.appendChild(muteBtn)`, CSS `position:fixed; top:0; right:0` (44×44 원형).
  → **무대(`#stage`) 바깥의 뷰포트 고정 요소**. fitStage 스케일 영향 안 받음.
- 오디오 모듈(`audio.toggleMute/onMuteChange/isMuted`, `hg_muted` localStorage)은
  각 페이지 상단에 있고 이번 작업에서 **손대지 않는다**. 팝업의 음소거 행이 그대로 재사용.
- 교사용 히든 PIN 점프 메뉴(코너 롱프레스)는 **별개 기능** — 그대로 공존, 손대지 않음.
- 앱 종료 API: `window.Capacitor?.Plugins?.App?.exitApp()`. 이미 각 페이지 "R5 하드웨어
  뒤로가기 무시" 블록이 `window.Capacitor.Plugins.App` 을 참조 중. 네이티브(APK)에서만
  동작, 브라우저 dev 에선 no-op(가드로 안전).

## 핵심 설계 결정

### 1. 무대(`#stage`) 안 배치 (기존 mute 버튼과 다른 점)

기존 음소거 버튼은 무대 바깥 뷰포트 고정이었다. 이번 UI는 **무대 안에 px 절대배치**한다
(`position:absolute; top/right`, `#stage` 자식). 이유:

- 게임 프레임에 소속 → dev 브라우저에서 레터박스 검은 띠가 생겨도 버튼이 게임 코너에 붙음.
- 팝업 dim이 무대 영역만 덮어 시각적으로 깔끔.
- `무대 내부는 px 절대배치` 규약과 일관.

실기기(immersive, 무대가 화면을 정확히 채움)에선 무대 우측상단 = 화면 우측상단이라 결과 동일.

### 2. 무대 계열별 두 크기 세트 (배율차 정확히 1.5배)

| 페이지 계열 | 무대 | fitStage 배율 |
|---|---|---|
| intro · auth · home · guide | 1280×800 | ×1.0 |
| 미션(행성) 페이지 | 1920×1200 | ×0.667 |

무대 안 요소는 배율을 같이 받으므로, 물리 크기를 맞추려면 **1920 계열 = 1280 계열 × 1.5**
(= 1280 계열은 1920 계열 × 2/3). playwright 검증 후 확정한 최종 크기:

| 요소 | 1280 무대 값 | 1920 무대 값 | 실기기 물리 크기 |
|---|---|---|---|
| 메뉴 버튼 ☰ | 40×40 (폰트 20) | 60×60 (폰트 30) | ≈40px |
| 팝업 버튼(음소거/종료) | 높이 64, 폰트 23 | 높이 96, 폰트 34 | 중간 |
| 팝업 패널 폭 | 350 | 520 | 중앙 |

메뉴 버튼은 눈에 띄되 게임을 가리지 않게 작게, 실제 UI는 팝업이 담당. (초기엔 더 크게 잡았다가
playwright 스크린샷 확인 후 ×2/3 로 축소 확정.)

## 컴포넌트 상세

### 메뉴 버튼 `.hg-menu-btn`
- `#stage` 자식, `position:absolute; top:<여백>; right:<여백>`, 원형, 텍스트 `☰`.
- 기존 mute-btn 관례 계승: `data-sfx="none"`(있다면), 반투명 배경, hover/focus 시 불투명.
- 클릭 → 팝업 open.

### 팝업 `.hg-menu-overlay` + `.hg-menu-panel`
- 오버레이: `#stage` 자식, 무대 전체 덮는 dim(반투명 배경), 중앙 정렬.
- 패널: 큰 버튼 세로 나열(태블릿·아동용).
  - `🔊 소리 끄기` / `🔇 소리 켜기` — 토글. `audio.isMuted()`로 초기 라벨, `audio.toggleMute()`
    호출, `audio.onMuteChange`로 라벨 갱신(팝업 안에서 즉시 반영).
  - `🚪 앱 종료` — 클릭 시 확인 뷰로 전환.
  - 닫기(×) 버튼 + **배경(dim) 탭 = 닫기**.
- 모든 텍스트 요소: `white-space: pre-line` + `word-break: keep-all`.
- **무대 입력 차단**: 메뉴 버튼·오버레이의 `click`/`pointerdown` 에 `stopPropagation` — 팝업이
  떠 있는 동안(및 여닫는 클릭이) 무대의 탭-진행 핸들러(`stage.addEventListener("click", …)`)로
  버블링되지 않게 한다. dim 오버레이가 무대를 덮어 팝업 밖 게임 요소는 애초에 클릭되지 않는다.

### 앱 종료 확인 흐름
`🚪 앱 종료` 클릭 → 패널 내용이 확인 뷰로 전환:
- 문구 "앱을 종료할까요?"
- `종료` → `window.Capacitor?.Plugins?.App?.exitApp()` (네이티브만; 브라우저 no-op)
- `취소` → 첫 팝업 뷰로 복귀(또는 닫기)

### z-index
- 메뉴 버튼·팝업은 무대 내 페이지 콘텐츠보다 위. 무대 안 stacking 안에서 충분히 큰 z 사용
  (예: 버튼 9000, 팝업 오버레이 9500). planet 페이지의 `.efg-overlay z:100` 등보다 위.
- 교사용 히든 메뉴(무대 바깥 전역)와는 별개 스택.

## 구현 위치

- 각 페이지 `script.js`의 "공통 블록 3-② 음소거 버튼" 블록을
  **"공통 블록 3-② 메뉴 버튼 + 팝업"** 으로 교체(기존 `.mute-btn` 생성·append 제거).
- DOM 부착: `document.getElementById("stage").appendChild(...)` (body append 아님).
- 대응 CSS를 각 페이지 `style.css` 의 mute-btn CSS 자리에 교체 복사.

- 1920 무대 계열의 reference. 여기서 버튼·팝업·종료 확인·음소거 이관을 완성하고 검증.
- 검증: playwright MCP로 실제 실행 — 버튼 표시 / 팝업 열림·닫힘(× + 배경 탭) /
  음소거 토글 라벨 반영 / 종료 확인창 표시(브라우저에선 exitApp no-op 확인) / 콘솔 에러 0 /
  스크린샷. 크기 미세 조정.

### Phase 2 — 나머지 22개 페이지로 확산
- Phase 1에서 확정된 블록(1920 값)을 복사하되, **1280 계열 페이지(intro·auth·home·guide)는
  1280 크기 세트(버튼 60 / 팝업버튼 96 / 패널 520)로 교체**하고, 미션 계열 페이지는 Phase 1과
  동일한 1920 세트(90 / 144 / 780)를 그대로 사용.
- 대상: intro(`www/`) · auth · home · guide · 각 planet prologue/mission (planet3/mission23 3D 포함).
- 대표 페이지 재검증(intro · auth · home · 미션 1개 · mission23).

### 보류(별도 후속)
- **Home 화면 기존 팝업(탐험 일지 등)과 겹칠 가능성** — 일단 만들고 레이아웃 조정.

## 범위 밖 (YAGNI)
- 팝업에 "홈으로" 등 추가 항목 없음(음소거·종료 딱 둘).
- 오디오 모듈 리팩터 없음.
- 교사용 히든 PIN 메뉴 변경 없음.
- 페이지 간 공유 파일화(assets/에 공통 js) 없음 — 규약상 페이지별 복사 유지.

## 성공 기준
- 전 페이지 우측 상단 ☰ 표시, 무대 안 고정, 이전보다 크게.
- 클릭 시 팝업 → 음소거 토글 정상 동작(기존과 동일), 앱 종료(확인 경유) 배선.
- 기존 독립 음소거 버튼 제거됨(중복 없음).
- 콘솔 에러 0, 대표 페이지 playwright 검증 통과.
