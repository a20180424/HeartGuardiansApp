# 사운드 튜닝 가이드

이 앱의 소리는 **전부 오실레이터로 합성**한다(오디오 파일 0개 → APK 크기 영향 없음).
그래서 튜닝은 에셋 교체가 아니라 **숫자 몇 개를 고치는 일**이다. 이 문서는 그 손잡이 위치를 모은다.

설계 배경은 [스펙](superpowers/specs/2026-07-17-sound-design-design.md) 참조.

> **전부 사람 귀로 맞췄다.** 말풍선 타자기(파형·엔벨로프·주파수)와 효과음 레벨(gain)
> 모두 사운드 벤치에서 번갈아 들으며 정한 값이다 — 종이 위 추정치가 아니다.
>
> **숫자가 들쭉날쭉한 게 정상이다.** "같은 gain = 같은 크기"가 아니기 때문(아래 참조).

## 이 앱에 없는 것 (시도했다가 뺌)

| 무엇          | 왜 뺐나                                                                       |
| ------------- | ----------------------------------------------------------------------------- |
| 배경음악(BGM) | 어울리는 음원을 못 찾음. 곡·재생기는 그대로 있고 `BGM_ENABLED = false`로 꺼둠 |
| 앰비언스 패드 | BGM 을 합성 패드로 우회하려 했으나 **초등학생에겐 있으나 마나** 한 수준이었다 |
| 씬 전환음     | 220ms 전환에 어울리는 소리를 못 만듦 (아래 "소리 목록" 절 참조)               |

세 가지 모두 **사람이 직접 들어보고 내린 결정**이다. 되살리려면 아래 각 절의 경고를 볼 것.

## 사운드 벤치 (청취 도구)

`tools/sound-bench.html` — **15개 소리 전부에 gain·파형 노브**가 달린 브라우저 도구.
`audio.ts`의 `tone()`·`SOUNDS`와 `typeSound.ts`의 `blipAt`을 **그대로 복제**해서,
여기서 들리는 소리가 게임에서 나는 소리와 같다.

여는 법 — 둘 중 아무거나:

```bash
npm run dev     # → http://localhost:5173/tools/sound-bench.html
```

또는 파일을 브라우저로 직접 열어도 된다(서버 불필요).

**`dist/`에 안 들어간다** — vite 진입점이 `index.html` 하나뿐이라 APK에 실리지 않는다.

쓰는 법: 노브를 옮기면 **즉시 그 소리가 재생**된다. 마음에 들면 맨 아래 **📋 복사**를 눌러
나온 내용을 그대로 전달하면 코드에 반영한다. 코드값에서 바뀐 항목은 노란색.

**말풍선 타자기는 진짜 게임 대사(`mission01.json`)를 진짜 속도(30ms/글자)로 실연한다** —
blip 은 단발로 들으면 판단이 안 되기 때문이다.

⚠️ **벤치는 코드의 스냅샷이다.** `audio.ts`의 값을 바꾸면 벤치의 기본값도 같이 고쳐야
"바뀐 항목 없음"이 참이 된다.

## 말풍선 타자기 blip (가장 튜닝이 필요한 곳)

미션 대사가 30ms마다 한 글자씩 찍힌다(= 초당 33회). 매 글자 소리를 내면 겹쳐서 톱질 소음이 되므로
성글게 울리고 공백·문장부호는 건너뛴다.

**아래는 사람이 사운드 테스트 벤치에서 실제 게임 대사를 실제 속도로 들으며 맞춘 값이다**(추정치 아님).

| 손잡이        | 파일                     | 현재       | 의미                                  |
| ------------- | ------------------------ | ---------- | ------------------------------------- |
| `BLIP_EVERY`  | `src/lib/typeSound.ts:9` | `4`        | 어절 안에서 몇 음절마다 울릴지        |
| `type` (파형) | `src/lib/audio.ts`       | `triangle` | **선명함의 가장 큰 레버** (아래 참조) |
| `gain`        | `src/lib/audio.ts`       | `0.2`      | `tap`(0.16)보다 조금 크게             |
| `freq` 하티   | `src/lib/audio.ts`       | `500`      | 가이드 하티 — 낮은 쪽                 |
| `freq` 친구   | `src/lib/audio.ts`       | `600`      | 감정을 말하는 외계인 — 높은 쪽        |
| `attack`      | `src/lib/audio.ts`       | `0.01`     | 짧을수록 "탁" 때리는 느낌             |
| `dur`         | `src/lib/audio.ts`       | `0.05`     | 짧으면 틱/클릭, 길면 음(音)           |
| `release`     | `src/lib/audio.ts`       | `0.04`     | 꼬리. 짧으면 딱 끊겨 또렷             |

**파형이 “뭉툭함”을 좌우한다.** 처음엔 `sine`이었는데 **배음이 하나도 없어 물렁하게** 들렸다.
배음이 늘수록 밝고 선명해진다: `sine`(배음 0) → `triangle`(약한 홀수 배음, **현재**) →
`square`(강한 홀수 배음, 8비트) → `sawtooth`(모든 배음, 가장 날카로움).
자주 울리는 소리라 `square` 이상은 자극적일 수 있다.

**⚠️ blip 하나의 실제 길이 = `dur + release + 0.02` = 110ms.** 그런데 blip 간격은 어절 길이에
달렸고, 1음절 어절(예: "수 있어"의 "수") 뒤에는 **60ms 만에 다음 blip이 온다** → 겹친다.
지금은 사람이 그 상태로 듣고 고른 값이지만, `dur`/`release`를 더 늘리면 겹침이 심해진다.

**`BLIP_EVERY`가 실제로 뜻하는 것:** 공백·문장부호에서 카운터가 **리셋**되므로(쉼 뒤 첫 글자는 늘
울리게), 한국어에선 사실상 **어절 첫 음절마다** 울린다. 한국어 어절이 대부분 2~3음절이라
**3과 4는 긴 단어(4음절 이상)에서만 갈리고, 5 이상은 4와 사실상 같다.** 실질적인 선택지는
`2`(어절마다 두 번, 촘촘) 대 `3~4`(어절 첫 음절 위주, 성글게)뿐이다.

**⚠️ 함정 — `BLIP_EVERY = 1`은 완전 무음이다.** 판정식이 `n % BLIP_EVERY === 1`인데 `n % 1`은
항상 0이라 절대 참이 안 된다. "1이면 매 글자 울리겠지"는 틀렸다.

**⚠️ `BLIP_EVERY`를 바꾸면 테스트 2건이 깨진다** — `typeSound.test.ts`의 "쉼 없는 긴 단어…"와
"여러 단어로 된 대사…"가 그 상수에 묶여 있다(파일 안에 주석으로 표시해 둠). 기대값을 손으로
짐작하지 말고 구현으로 계산할 것.

## 앰비언스 패드 — 만들었다가 뺐다

BGM 음원을 못 찾은 걸 우회하려고 멜로디 없는 저음 패드(hub `[110, 164.81]` / planet
`[98, 146.83]`, 완전5도)를 합성해 존별로 깔았었다. 논리는 맞았지만 — 곡이 아니라 공간감이라
"어울리는가" 판단이 필요 없고 APK도 0바이트 — **정작 초등학생에겐 있으나 마나 했다.**
그래서 뺐다(`Ambience.tsx` 삭제, `audio.ts`의 `startPad`/`stopPad` 제거).

되살리려면 git 이력에서 꺼내면 된다. 다만 **같은 함정을 다시 밟지 말 것**: 저음은 사람 귀가
둔한 대역이라 0.05 같은 값으로는 태블릿 스피커에서 거의 안 들린다. 크게 하면 효과음을 덮는다.

`src/lib/zone.ts`(경로 → `hub`/`planet`/`silent` 판별)는 **남아 있다** — `bgm.tsx`가 쓴다.
원래 거기 있던 걸 앰비언스와 공유하려고 추출한 것이라, 앰비언스가 없어져도 제자리로 돌아간 셈.

## 버튼 효과음

App 레벨 리스너 하나가 모든 버튼을 처리한다(`src/App.tsx`). 미션 UI는 전부 `#stage` 안이라
`closest("#stage")`로 제외된다 — 미션은 자체 사운드 25곳이 이미 있다.

버튼별 소리는 **선언**한다:

```text
<button>                  → tap  (기본)
<button data-sfx="pop">   → pop  (팝업 열기)
<button data-sfx="none">  → 무음
```

매핑 규칙은 `src/lib/uiSfx.ts`. 오타(`data-sfx="popp"`)는 무음이 아니라 기본 `tap`으로 떨어진다 —
조용히 사라지는 것보다 눈에 띄는 게 낫기 때문.

`"none"`은 지금 셋뿐이고 이유가 각각 다르다: **음소거 버튼**(음소거하며 tap 이 나면 앞뒤가 안
맞음), **인트로·아웃트로의 화면 전체 탭 레이어**(영상 보며 아무데나 누를 때마다 영화 위에
비프음이 겹침 — 건너뛰기 버튼은 진짜 버튼이라 소리를 유지한다).

## 소리 목록 (주파수 = 음 높이)

`src/lib/audio.ts`의 `SOUNDS`. 디버깅 시 주파수로 어떤 소리인지 특정할 수 있다.

| 소리         | 주파수(Hz)                 | gain      | 파형     | 쓰이는 곳                   |
| ------------ | -------------------------- | --------- | -------- | --------------------------- |
| `tap`        | 520                        | 0.16      | sine     | 버튼 기본 + 미션 탭         |
| `pop`        | 400→680                    | 0.16      | triangle | 팝업 열기                   |
| `title`      | 523·659·784·1047           | 0.13      | triangle | 로그인 성공, 미션 인트로    |
| `wrong`      | 320→200                    | 0.15      | sine     | 로그인 실패, 미션 오답      |
| `blipHati`   | 500                        | 0.2       | triangle | 하티 타자기                 |
| `blipFriend` | 600                        | 0.2       | triangle | 친구 타자기                 |
| `whoosh`     | 520→150                    | 0.06      | sawtooth | **미션 전용** (카드가 스륵) |
| `select`     | 600·900                    | 0.14/0.12 | triangle | 미션: 카드 고르기           |
| `drop`       | 880·1320                   | 0.16/0.13 | sine     | 미션: 카드 놓기             |
| `correct`    | 660·880·1175               | 0.15      | triangle | 미션: 정답                  |
| `stage`      | 600·900                    | 0.12      | triangle | 미션: 스테이지 전환         |
| `sparkle`    | 1600~2700 랜덤 ×6          | 0.07      | triangle | 미션: 반짝임                |
| `recover`    | 200→900 + sparkle          | 0.11      | sawtooth | 미션: 회복                  |
| `reveal`     | 784·988·1318               | 0.1       | triangle | 미션: 공개                  |
| `fanfare`    | 523·659·784·1047 + sparkle | 0.14      | triangle | 미션: 완료 팡파르           |

**gain 이 들쭉날쭉한 건 맞춘 결과다.** 숫자를 통일하면 오히려 안 맞는다 — `sawtooth`는 모든
배음을 담아 같은 gain 의 `sine`보다 훨씬 크고(그래서 `whoosh` 0.06 이 그대로), `sparkle`은
6음이 겹쳐 합치면 사실상 0.42이며, 사람 귀는 2~4kHz에 예민하고 저음엔 둔하다. **번갈아 들으며
"같은 크기로 들리게" 맞춘 것**이고, 다시 조정할 때도 숫자가 아니라 귀를 기준으로 할 것.

**`sparkle`을 바꾸면 `recover`·`fanfare`도 같이 바뀐다** — 그 둘이 sparkle 을 덧붙여 부른다.

**씬 전환에는 소리가 없다 — 시도했다가 뺐다.** 처음엔 기존 `whoosh`를 재사용했는데, 그 값
(0.06)은 원래 _미션 안에서 카드가 스륵 지나갈 때_ 쓰라고 잡힌 것이라 화면이 통째로 바뀌는
순간엔 너무 작았다. 전용 변종(`whooshNav`)까지 만들었지만 끝내 어울리는 소리를 못 찾았다 —
**전환이 220ms(`FADE_MS 160` + `HOLD_MS 60`)라 짧으면 뚝 끊기고, 2~3초 스웰로 가면 화면이
바뀐 뒤에도 울려 다음 씬을 침범한다.** 버튼 `tap` 만으로 충분하다는 결론.

⚠️ **다시 넣는다면 `data-sfx="none"`을 함께 봐야 한다.** 전환음이 있던 시절엔 전환 버튼 6곳에
`data-sfx="none"`을 붙여 `tap`+전환음 이중 발생을 막았다. 전환음을 없애면서 그 속성도 걷어냈다
(안 그러면 버튼이 벙어리가 된다). 전환음을 되살리면 그 6곳을 다시 무음 처리해야 한다:
`PlanetButton` / `WelcomePanel` / `PrologueTemplate` / `intro` 시작하기 / `outro` 홈으로 /
`HiddenMenuOverlay` ×2.

**소리를 재사용할 때는 "같은 소리인가"만 보지 말고 "같은 크기의 사건인가"도 봐야 한다** —
이 건의 교훈이다.

**미션 소리도 이번에 같이 바뀌었다.** 원래 계획은 "기존 미션 사운드 무변경"이었지만, 레벨을
통일하려면 `tap`·`pop`·`title`·`wrong`·`sparkle`(미션도 쓰는 소리)을 건드릴 수밖에 없어서
사람이 그 제약을 의도적으로 뒤집었다. 미션 안에서 이 소리들이 예전보다 크다.

## 알려진 이슈 — 음소거 버튼 겹침

음소거 버튼은 App 레벨 `position: fixed`(무대 밖)이고, 좌상단은 히든 메뉴 진입 제스처
(`useCornerLongPress.ts` `CORNER_PX = 200`) 자리라 쓸 수 없어 우상단에 있다.
1280×800(= APK 조건)에서 측정한 현재 상태:

| 화면            | 상태                                                    |
| --------------- | ------------------------------------------------------- |
| 인트로          | 겹침 없음 ✅ (`.intro__skip`을 `top: 56px`로 내림)      |
| **아웃트로**    | **28×28px 겹침** — 건너뛰기 클릭을 음소거가 가져간다    |
| **홈 학습목표** | **26×26px 겹침** — 배너 우상단 클릭을 음소거가 가져간다 |
| 미션            | 겹침 없음 (우상단·우하단 모두 비어 있음)                |

아웃트로는 `.outro__skip`이 `src/scenes/outro/Outro.css`에 **따로** 있어서 인트로만 고치면
따라오지 않는다. 인트로와 같이 `top`을 내리면 해결된다.

**⚠️ 브라우저에서는 재현되지 않는다.** 건너뛰기·배너는 무대(FixedStage) 안이라 letterbox 시
같이 축소되고, 음소거는 무대 밖 `position: fixed`라 실뷰포트에 고정된다. 창 크기가
1280×800이 아니면 둘이 저절로 떨어져 보인다. **확인하려면 창을 정확히 1280×800으로
맞추거나 APK로 봐야 한다.**

## 소리를 어떻게 검증했나 (자동화의 한계)

자동화는 **소리를 들을 수 없다.** 대신 `AudioParam.prototype.setValueAtTime`을 후킹해
오실레이터에 예약되는 주파수를 기록하면, `[520, 320]` 같은 지문으로 "탭 + 오답이 났다"를 특정할 수 있다.
그래서 "의도한 순간에 의도한 소리가 난다"는 증명됐지만, **음량·음색·거슬림은 전부 미검증**이다.

계측 시 함정 두 가지:

- `osc.frequency.value`는 항상 **440**(기본값)을 돌려준다 — `tone()`이 `setValueAtTime`으로 *예약*만 하기 때문.
  예약을 후킹해야 한다.
- 해시만 다른 URL로 이동하는 것은 **리로드가 아니다**. 후킹이 살아남아 중첩되면 소리가 몇 배로 세어진다.
