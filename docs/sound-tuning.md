# 사운드 튜닝 가이드

이 앱의 소리는 **전부 오실레이터로 합성**한다(오디오 파일 0개 → APK 크기 영향 없음).
그래서 튜닝은 에셋 교체가 아니라 **숫자 몇 개를 고치는 일**이다. 이 문서는 그 손잡이 위치를 모은다.

설계 배경은 [스펙](superpowers/specs/2026-07-17-sound-design-design.md) 참조.

> **중요:** 아래 값들은 전부 **귀로 검증되지 않은 종이 위 추정치**다.
> 코드·계측으로는 "의도한 순간에 의도한 소리가 난다"까지만 확인했다.
> "그 소리가 듣기 좋은가"는 사람만 판단할 수 있다.

## 전체 끄기 스위치

| 무엇          | 파일                   | 상수                                                          |
| ------------- | ---------------------- | ------------------------------------------------------------- |
| 배경음악(BGM) | `src/lib/bgm.tsx`      | `BGM_ENABLED = false` — 어울리는 음원을 못 찾아 꺼둠          |
| 앰비언스 패드 | `src/lib/Ambience.tsx` | `AMBIENCE_ENABLED = true` — 마음에 안 들면 `false` 한 줄로 끝 |
| 전체 음소거   | 런타임                 | 우상단 🔊 버튼. `localStorage`의 `hg_muted`에 저장            |

## 마스터 볼륨

`src/lib/audio.ts`의 `VOL = 0.5`. 모든 소리가 이 게인을 통과하므로 **전체가 크거나 작을 때** 여기를 만진다.
개별 소리의 균형이 문제면 아래 각 항목을 만진다.

## 말풍선 타자기 blip (가장 튜닝이 필요한 곳)

미션 대사가 30ms마다 한 글자씩 찍힌다(= 초당 33회). 매 글자 소리를 내면 겹쳐서 톱질 소음이 되므로
**말하는 글자 3개마다 한 번**만 울리고 공백·문장부호는 건너뛴다. 실측 간격 약 103ms(초당 9.7회).

| 손잡이       | 파일                       | 현재    | 의미                                         |
| ------------ | -------------------------- | ------- | -------------------------------------------- |
| `BLIP_EVERY` | `src/lib/typeSound.ts:9`   | `3`     | 크게 하면 더 뜸하게(4~5로 올리면 차분해짐)   |
| `gain`       | `src/lib/audio.ts:193-194` | `0.04`  | 기존 `tap`(0.11)의 약 1/3. 거슬리면 0.03으로 |
| `freq` 하티  | `src/lib/audio.ts:193`     | `320`   | 가이드 하티 — 낮고 차분                      |
| `freq` 친구  | `src/lib/audio.ts:194`     | `480`   | 감정을 말하는 외계인 — 조금 높게             |
| `dur`        | `src/lib/audio.ts:193-194` | `0.025` | 25ms. 길게 하면 서로 겹치기 시작한다         |

**알아둘 것:** 공백·문장부호에서 카운터가 리셋되므로, 한국어에서는 사실상 **어절 첫 음절마다 울린다**
(2–3음절 단어가 대부분이라). 스펙 문구 "2~3글자마다"의 문자적 해석과는 다르지만 말 리듬에 더 가깝다.
계획서의 원래 구현은 이 두 요구를 동시에 만족할 수 없는 모순이었고, 리셋 방식이 유일한 해법이었다.

## 앰비언스 패드

멜로디 없는 저음 패드. 곡이 아니라 "공간감"이라 어울리는 음원을 찾을 필요가 없다 —
BGM이 막힌 지점을 우회하는 장치다.

| 손잡이      | 파일                         | 현재                                        | 의미                                                      |
| ----------- | ---------------------------- | ------------------------------------------- | --------------------------------------------------------- |
| 존별 주파수 | `src/lib/Ambience.tsx:13-14` | hub `[110, 164.81]` / planet `[98, 146.83]` | A2+E3 / G2+D3                                             |
| 음량        | `src/lib/audio.ts:111`       | `0.03`                                      | 효과음(0.11~0.16)보다 훨씬 아래여야 한다. 방해되면 낮춘다 |
| 페이드인    | `src/lib/audio.ts:111`       | `t0 + 3`                                    | 3초에 걸쳐 서서히                                         |
| 흔들림 속도 | `src/lib/audio.ts:113`       | `0.05 + i * 0.015`                          | LFO 주기 12~20초. 느릴수록 잔잔                           |
| 흔들림 깊이 | `src/lib/audio.ts:114`       | `0.015`                                     | 크면 출렁임이 커진다                                      |

존 판별은 `src/lib/zone.ts` — `hub`(로그인·홈) / `planet`(행성·미션) / `silent`(인트로·아웃트로, 동영상 자체 사운드).

**개발 중 참고:** StrictMode 때문에 `npm run dev` 최초 로드 직후 ~1.6초 동안 패드가 두 벌 겹쳐 살짝 크게 들린다.
프로덕션 빌드에는 없는 현상이니 APK에서 판단할 것.

## 버튼 효과음

App 레벨 리스너 하나가 모든 버튼을 처리한다(`src/App.tsx`). 미션 UI는 전부 `#stage` 안이라
`closest("#stage")`로 제외된다 — 미션은 자체 사운드 25곳이 이미 있다.

버튼별 소리는 **선언**한다:

```text
<button>                  → tap  (기본)
<button data-sfx="pop">   → pop  (팝업 열기)
<button data-sfx="none">  → 무음 (씬 전환음이 이미 우는 버튼)
```

매핑 규칙은 `src/lib/uiSfx.ts`. 오타(`data-sfx="popp"`)는 무음이 아니라 기본 `tap`으로 떨어진다 —
조용히 사라지는 것보다 눈에 띄는 게 낫기 때문.

## 소리 목록 (주파수 = 음 높이)

`src/lib/audio.ts`의 `SOUNDS`. 디버깅 시 주파수로 어떤 소리인지 특정할 수 있다.

| 소리         | 주파수(Hz)             | 쓰이는 곳                |
| ------------ | ---------------------- | ------------------------ |
| `tap`        | 520                    | 버튼 기본                |
| `pop`        | 400                    | 팝업 열기                |
| `whoosh`     | 520→150                | 씬 전환                  |
| `wrong`      | 320→200                | 로그인 실패, 미션 오답   |
| `title`      | 523·659·784·1047       | 로그인 성공, 미션 인트로 |
| `blipHati`   | 320                    | 하티 타자기              |
| `blipFriend` | 480                    | 친구 타자기              |
| 앰비언스     | 110·164.81 / 98·146.83 | hub / planet             |

나머지(`select` `drop` `correct` `stage` `sparkle` `recover` `reveal` `fanfare`)는 미션 전용이며
이번 작업에서 건드리지 않았다.

## 알려진 이슈

- **홈에서 음소거 버튼이 "학습 목표" 배너 우상단 모서리를 ~38×38px 덮어 그 부분 클릭을 가져간다.**
  좌상단은 히든 메뉴 진입 제스처(`useCornerLongPress.ts` `CORNER_PX = 200`) 자리라 쓸 수 없다.
  미션 화면은 우상단·우하단 모두 비어 있어 무관. 순수 CSS 문제라 `src/lib/mute-button.css`의
  `top`/`right` 또는 `.home-goal` 쪽을 한 줄 조정하면 된다.

## 소리를 어떻게 검증했나 (자동화의 한계)

자동화는 **소리를 들을 수 없다.** 대신 `AudioParam.prototype.setValueAtTime`을 후킹해
오실레이터에 예약되는 주파수를 기록하면, `[520, 320]` 같은 지문으로 "탭 + 오답이 났다"를 특정할 수 있다.
그래서 "의도한 순간에 의도한 소리가 난다"는 증명됐지만, **음량·음색·거슬림은 전부 미검증**이다.

계측 시 함정 두 가지:

- `osc.frequency.value`는 항상 **440**(기본값)을 돌려준다 — `tone()`이 `setValueAtTime`으로 *예약*만 하기 때문.
  예약을 후킹해야 한다.
- 해시만 다른 URL로 이동하는 것은 **리로드가 아니다**. 후킹이 살아남아 중첩되면 소리가 몇 배로 세어진다.
